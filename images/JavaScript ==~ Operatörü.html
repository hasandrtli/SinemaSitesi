Hocam, JavaScript'te ==~ diye bir operatör yok. Ama benzer görünen birkaç şeyin karışmış olma ihtimali var. Aşağıda olası anlam karışıklıklarını açıklayayım:

1. == ve === (Karşılaştırma Operatörleri)

  • ==: Gevşek eşitlik (tip dönüşümü yapar).

  • ===: Sıkı eşitlik (tip dönüşümü yapmaz).

js
5 == '5'   // true (tip dönüşümü yapılır)
5 === '5'  // false (tip farklı)

2. ~ (Bit düzeyinde TERSLEME operatörü)

Bu tek başına kullanılabilir:

js
~5  // -6 olur, çünkü bit düzeyinde terslenmiş halidir

Ama bazı durumlarda indexOf ile birlikte kısa yol olarak kullanılır:

js
if (~"hello world".indexOf("hello")) {
  console.log("Bulundu!");
}

  • indexOf bulamazsa -1 döner.

  • ~(-1) → 0 olur → falsy

  • Bulursa ~0 veya daha büyük → truthy

Bu yüzden bazı eski kodlarda bu tarz kısa yol yazımlar görülebilir.

3. ==~ Karışıklığı

Muhtemelen yanlışlıkla == ile ~'yi birleştirmiş olabilirsin, ama JavaScript bunu tanımaz ve syntax hatası verir.

js
if (a ==~ b)  // ❌ Hatalı, böyle bir operatör yok

Doğru Örnek:

Eğer amaç şuysa:

js
if (~str.indexOf("abc")) {
  // "abc" geçtiği sürece çalışır
}

ya da sade haliyle:

js
if (str.includes("abc")) {
  // modern ve okunabilir yöntem
}

Özet:

  • ==~ JavaScript’te yok.

  • == ayrı, ~ ayrı operatörlerdir.

  • Büyük ihtimalle ~indexOf(...) tarzı bir yapıyı veya == ile karışıklık olmuş olabilir.

JavaScript’te “eşit değilse” ifadesi için iki tane karşılaştırma operatörü vardır:

✅ 1. != — Gevşek eşit değil

Bu, değerleri karşılaştırır ama tip dönüşümüne izin verir:

js
5 != "5"   // false — çünkü değerler eşit sayılır (tip dönüşümü var)

✅ 2. !== — Sıkı eşit değil (tavsiye edilen)

Bu hem değeri hem tipi karşılaştırır. Genellikle !== kullanmak daha güvenlidir.

js
5 !== "5"  // true — çünkü tipler farklı (number vs string)
5 !== 5    // false — çünkü hem değer hem tip aynı

Örnek Kullanımlar

js
let a = 10;

if (a != "10") {
  console.log("a, '10'a eşit değil (tip dönüşümüne göre).");
}

if (a !== "10") {
  console.log("a, '10'a tam olarak eşit değil (tip dönüşümü yok).");
}

Kısacası:

AnlamOperatörAçıklama
Eşit değil (gevşek)!=Tip dönüşümü yapar
Eşit değil (sıkı)!==Tip dönüşümü yapmaz, tavsiye edilir
